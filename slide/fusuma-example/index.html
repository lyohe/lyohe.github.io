$ npm i fusuma -D

事前にスライド用のディレクトリを作って移動する

npm init しておくとよさそう

$ npx fusuma init

Fusuma でスタイルを変更する方法

  • 普通に CSS を書く
  • テーマを使う
  • CSS Variables を使う

普通に CSS を書く

https://hiroppy.github.io/fusuma/docs/guides/slide-syntax#adding-class-names

普通に CSS を書く

<!-- classes: foo -->

HTML コメントっぽい形式でスライド内にクラスを追加する。

style.css に CSS を書くと...

.foo {
  color: #3498db;
  background: #ccc;
}

このように反映される

block-start と block-end で div を作れる

同じページ内で一部に CSS を適用するときに便利

<!-- block-start: foo -->

ここに foo クラスを適用する

<!-- block-end -->

ここに foo クラスを適用する

block は div タグ

block でこういう階層も作れる

$ npx fusuma build

↓

$ npx fusuma pdf

slide.pdf が生成される

PDF 出力の注意点

先に build していないと export に失敗する

Build & Deploy

スライドを build して HTML を生成し、それを GitHub Pages に deploy できる。

$ npx fusuma deploy

Deploy で行われるのは

remote origin の gh-pages ブランチに build した dist/ 以下を入れる

QR コードも作れるよ

<!-- qr: https://lyohe.github.io/ -->

終わり

$ npx fusuma start-prod

これで prod を起動する
